# CodeLeap Network

🚀 **Live:** [https://code-leap-challenge-tau.vercel.app](https://code-leap-challenge-tau.vercel.app)

A full-stack social network application where authenticated users can create, edit, and delete posts. Posts from an external API are also fetched and displayed alongside local ones. Built as a challenge project to practice modern full-stack development with Next.js App Router, authentication, ORM, and testing.

## ✔️ Techniques and technologies used  
- **Next.js**: App Router framework used for server and client components, API routes, and server-side authentication redirects.  
- **React**: UI library used for building interactive client components with state management.  
- **TypeScript**: Used throughout the project to add static typing, increasing code reliability and developer experience.  
- **Tailwind CSS**: Used for styling and building a fully responsive layout efficiently with utility classes.  
- **Shadcn UI**: Used to build accessible and modern UI components such as Button, Dialog, Input, and Textarea.  
- **Prisma**: ORM used for database schema definition, migrations, and type-safe queries.  
- **Neon**: Serverless PostgreSQL database provider used to host and manage the project data in the cloud.  
- **Kinde**: Used to implement the full authentication flow including login, register, logout, session management, and route protection.  
- **SVG Icons**: Custom SVG icons used for edit and delete actions inside post cards.  
- **Full Responsive**: Layout fully adapted for mobile, tablet, and desktop screens.  
- **Jest**: Testing framework used to write and run unit tests for utility functions, UI components, and API routes.  
- **Husky**: Used to configure a pre-push Git hook that automatically runs all tests before every push, blocking it on failure.  
- **Git/GitHub**: Tools used for version control, code hosting, and triggering automatic deployments.  
- **Vercel**: Platform used to deploy and host the project in production with automatic deploys on push.  

## ⚠️ Challenges  
Integrating posts from an external API alongside local database posts required careful ID collision handling and ensuring external posts are displayed as read-only. Configuring the global Roboto font with Next.js font variables and Tailwind CSS v4 required precise mapping between CSS custom properties. Setting up Jest with Next.js App Router involved separating test environments (jsdom vs node) per test file and correctly mocking Prisma and Kinde dependencies.

# CodeLeap Network

🚀 **Produção:** [https://code-leap-challenge-tau.vercel.app](https://code-leap-challenge-tau.vercel.app)

Aplicação full-stack de rede social onde usuários autenticados podem criar, editar e deletar posts. Posts de uma API externa também são buscados e exibidos junto com os locais. Construído como um projeto desafio para praticar desenvolvimento full-stack moderno com Next.js App Router, autenticação, ORM e testes.

## ✔️ Técnicas e tecnologias utilizadas  
- **Next.js**: Framework com App Router utilizado para componentes server e client, rotas de API e redirecionamentos de autenticação no servidor.  
- **React**: Biblioteca de UI utilizada para construir componentes client interativos com gerenciamento de estado.  
- **TypeScript**: Utilizado em todo o projeto para adicionar tipagem estática, aumentando a confiabilidade do código e a experiência de desenvolvimento.  
- **Tailwind CSS**: Utilizado para estilização e construção de layout totalmente responsivo com classes utilitárias.  
- **Shadcn UI**: Utilizado para construir componentes de UI acessíveis e modernos como Button, Dialog, Input e Textarea.  
- **Prisma**: ORM utilizado para definição de schema, migrações e queries com tipagem segura.  
- **Neon**: Provedor de banco de dados PostgreSQL serverless utilizado para hospedar e gerenciar os dados do projeto na nuvem.  
- **Kinde**: Utilizado para implementar o fluxo completo de autenticação incluindo login, cadastro, logout, gerenciamento de sessão e proteção de rotas.  
- **SVG Icons**: Ícones SVG customizados utilizados nas ações de editar e deletar dentro dos cards de post.  
- **Full Responsive**: Layout totalmente adaptado para telas mobile, tablet e desktop.  
- **Jest**: Framework de testes utilizado para escrever e executar testes unitários de funções utilitárias, componentes de UI e rotas de API.  
- **Husky**: Utilizado para configurar um hook Git pre-push que executa automaticamente todos os testes antes de cada push, bloqueando-o em caso de falha.  
- **Git/GitHub**: Ferramentas utilizadas para controle de versão, hospedagem de código e acionamento de deploys automáticos.  
- **Vercel**: Plataforma utilizada para publicar e hospedar o projeto em produção com deploys automáticos a cada push.  

## ⚠️ Desafios  
Integrar posts de uma API externa junto com os posts locais do banco de dados exigiu tratamento cuidadoso de colisão de IDs e garantia de que os posts externos sejam exibidos como somente leitura. Configurar a fonte Roboto globalmente com as variáveis de fonte do Next.js e Tailwind CSS v4 exigiu mapeamento preciso entre propriedades CSS customizadas. Configurar o Jest com o App Router do Next.js envolveu separar os ambientes de teste (jsdom vs node) por arquivo de teste e mockar corretamente as dependências do Prisma e do Kinde.
